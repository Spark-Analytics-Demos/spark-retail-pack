# spark_retail_pack — OSS dbt Core

The open-source dbt transformation layer for the Spark Retail Pack. Transforms raw Shopify, Stripe, GA4, Meta Ads, and Klaviyo data into a canonical dimensional model covering Sales Analytics, Customer 360, and Inventory Health.

## Prerequisites

- Python 3.11 or 3.12
- A Snowflake account with the service accounts provisioned (see `../setup/snowflake/`)
- A `.env` file at the repo root with credentials (see `.env` in repo root — never commit it)

## Setup

```bash
# 1. Create and activate a virtual environment (from this directory)
python -m venv .venv
.venv\Scripts\activate        # Windows
# source .venv/bin/activate   # macOS/Linux

# 2. Install dependencies
pip install -r requirements.txt

# 3. Configure your dbt profile
cp profiles.yml.template profiles.yml
# Edit profiles.yml — fill in the dev target credentials from your .env file

# 4. Install dbt packages
dbt deps
```

---

## Running dbt

```bash
# Validate the project parses correctly (no Snowflake connection needed)
dbt parse

# Run all models (dev target by default — writes to dev_<your_name>_* schemas)
dbt build

# Run only models you've changed, plus their downstream dependencies
dbt build --select state:modified+

# Run tests only
dbt test

# Check source freshness (requires live data in RAW_RETAIL.*)
dbt source freshness
```

---

## Generating docs

```bash
# Generate docs (requires Snowflake connection to build catalog)
dbt docs generate

# Serve locally at http://localhost:8080
dbt docs serve
```

The published docs site is auto-generated by the `dbt-docs` GitHub Actions workflow on every push to `main` and hosted on GitHub Pages.

## Conventions

All coding conventions, naming rules, materialization defaults, and test requirements are in [`CLAUDE.md`](../CLAUDE.md) at the repo root. Read it before writing any model.

Key rules:
- Staging: `stg_<source>__<table>.sql` (two underscores)
- Every model has a `schema.yml` with description, columns, and tests
- Every core and mart model has the 8-column audit footer (`add_audit_columns` macro)
- PII columns use the `pii_mask` macro — no plaintext PII in gold/mart layers
- Never add Pro features (Section 11) to this package
